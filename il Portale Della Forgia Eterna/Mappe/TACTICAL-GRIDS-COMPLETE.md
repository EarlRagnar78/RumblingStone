## üìê GRID OVERLAY SPECIFICATIONS

### Creating Physical Maps:

**For Printing:**
1. Export images 300 DPI minimum
2. Overlay transparent grid layer (1.5m squares)
3. Number squares alpha-numeric (A1-ZZ99)
4. Print large format (A3 minimum, A1 ideal)
5. Laminate for dry-erase markers

**For VTT (Virtual Tabletop):**
1. Import PNG/JPG to Roll20/Foundry
2. Align grid tool (set 1.5m per square)
3. Lock background layer
4. Add token layer above
5. Use fog of war for unexplored areas

**Grid Numbering System:**
- Horizontal: Letters (A-Z, then AA-AZ, BA-BZ, etc.)
- Vertical: Numbers (1-999)
- Format: Letter(s) + Number (e.g., AG27)
- Origin: Always top-left corner = A1

### Miniature Scales:

**28-32mm (Standard D&D):**
- 1 square = 1 inch physical
- Medium creature = 1 square
- Large creature = 2√ó2 squares (4 squares)
- Huge creature = 3√ó3 squares (9 squares)
- Gargantuan = 4√ó4 squares (16 squares)
- Colossal = 6√ó6+ squares (36+ squares)

**15mm (Half-scale):**
- 1 square = 0.5 inch physical
- Use same grid, smaller minis

### Terrain Features:

**Cover Markers:**
- Light Cover (+2 AC): Outlined yellow
- Medium Cover (+4 AC): Outlined orange
- Heavy Cover (+6 AC): Outlined red
- Total Cover (+8 AC): Outlined black
- Concealment (20-50%): Blue tint

**Elevation Markers:**
- +1.5m (1 square): Small "1" token
- +3m (2 squares): Small "2" token
- +4.5m (3 squares): Small "3" token
- Continue as needed

**Hazard Markers:**
- Fire: Red triangle
- Acid: Green circle
- Lava: Orange/red waves
- Pit: Black square with depth number
- Difficult Terrain: Brown hash marks

## üéØ TACTICAL TIPS FOR DMs

### Using Grid Maps Effectively:

**Preparation:**
- Pre-place enemy miniatures before session
- Use colored bases (red=enemy, blue=ally, green=neutral)
- Mark starting positions with tape/tokens
- Have elevation tokens ready (stackable dice work!)

**During Play:**
- Measure movement strictly (1 square = 1.5m = 5 feet)
- Use dice for elevation tracking (d6 under mini = 6m high)
- Allow players to plan with rulers (encourage tactics)
- Photograph setup for later reference/continuity

**Common Mistakes:**
- Forgetting diagonal movement (costs 1.5 moves per square)
- Ignoring elevation bonuses/penalties
- Not tracking AOE spell areas precisely
- Allowing "close enough" positioning (be exact!)

### Balancing for Different Party Sizes:

**3 Players:**
- Reduce enemy HP by 25%
- Remove 1-2 minions per encounter
- Increase treasure by 10%

**5 Players:**
- Increase enemy HP by 25%
- Add 2-3 minions per encounter
- Add hazards/complications

**6+ Players:**
- Use Advanced template on bosses (+4 all stats)
- Double minions
- Add secondary objectives (split party)
